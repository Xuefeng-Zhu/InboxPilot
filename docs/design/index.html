The queue lives at the top of the page while any document is still working. It disappears once everything is ready or failed.

State definitions
  • Queued — accepted by the server, waiting for a worker. No progress bar; the user knows it's in line.
  • Uploading — bytes in flight. Deterministic % bar, MB counter.
  • Processing — bytes are in. The system is chunking and embedding. Indeterminate bar; this is where the AI does the work.
Done & failed handoffs
  • When a document becomes Ready, it slides into the main list (Frame 03) and the queue count decrements.
  • If a document fails, it stays in this queue as a separate failed row (Frame 04) with the error inline.
Eng note
  • Polling: every 2s while queue > 0; switch to realtime once empty.
  • State machine: queued → uploading → processing → ready | failed.

Error message rules
  • Name what happened, not what broke. "Couldn't extract text" beats "PDFParseError".
  • Suggest the next step. The user shouldn't have to open a log to know what to try.
  • Show attempt count — "attempt 1 of 3" — so the user knows whether retry is worth it.
Server log link
  • Opens a new tab to the structured log for the failed job_id.
  • Include the job_id in both the inline callout and the log link so support can copy it.
Retry behavior
  • Optimistic — clicking "Retry" re-queues immediately, with a one-row flash of "Re-queued".
  • Auto-retry up to 3× on transient errors; beyond that, require a manual click.
Frame 05

Delete confirm — soft confirm with consequence copy

1 modal, 1 trigger

Delete Return policy.pdf?

This removes the document and its cached embeddings from the knowledge base. The AI will stop referencing it on the next message it sends.

The action is logged for your audit trail and can't be undone from here. If you want it back, you'll need to re-upload the file.

Customers won't see anything change until the AI's next reply.

Why this isn't "Are you sure?"
  • It names the specific document, so the user is never deleting a stranger.
  • It tells them what they get (gone) and what customers get (nothing, until next reply).
  • It says "can't be undone from here" — softer than "permanent", more honest than "destructive".
Button hierarchy
  • Cancel is a text link, not a button — the modal's safe action should be the default.
  • Delete document is the only filled button, in terracotta. No icon, no exclamation.
  • Enter triggers Cancel. Esc triggers Cancel. The "nuclear" option is always the second one.
Eng note
  • Soft-confirm, not password re-entry. This isn't a billing action.
  • On confirm, optimistically remove the row and show a toast with an Undo link (10s window).
  • Audit log: knowledge_document_deleted, with the title in metadata.

Body

How to request a return

Customers can request a return within 30 days of delivery for a full refund. The item must be unused and in its original packaging. To start a return, the customer should reply to their order confirmation email or message us here.

Refund timing

Refunds are issued to the original payment method within 5–7 business days of us receiving the returned item. The customer will get an email confirmation when the refund is on its way.

Non-returnable items

Gift cards, digital downloads, and perishable goods are not returnable. Custom orders can only be returned if they're defective on arrival…

Sidebar numbers
  • Chunks — count of embedded chunks. Mono font, monospace numbers.
  • Token estimate — rough total of tokens across chunks. Shown with a ~, not a promise.
  • Last tested — when the AI was last quizzed on this doc. (Same field as the list view.)
Source URL
  • For URL imports: shown as a clickable link, opens in a new tab.
  • For file uploads: shown as a dash. Don't render a broken "—" string as if it were a URL.
  • For URL imports, also show "Last fetched" so the user knows the snapshot is fresh (or not).
Test history
  • Timeline, not a table. Each entry is one line, with a dot in the status color.
  • Passed questions quote the chunk that answered them, so reviewers can audit fast.
  • Skipped questions are still listed — they're signal too.
